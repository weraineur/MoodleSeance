# Moodle Seance VS Code Extension

## Purpose
This extension scaffolds a pipeline to build e-learning content from documents.
It provides a sequence of commands: import, extract, confirm, send, generate, export.

## Files and configuration

### package.json
- Declares the extension metadata (name, publisher, version).
- Registers the VS Code commands used by the pipeline.
- Defines activation events so the extension loads only when a command runs.
- Adds build scripts to compile TypeScript.
- Command titles are short (Moodle: Import/Extract/Confirm/Send/Generate/Export).

### tsconfig.json
- Compiles TypeScript from `src` to `dist` using ES2020.
- Enables strict type checking for safer extension code.
- Produces source maps for easier debugging.

### .vscode/launch.json
- Launches a VS Code Extension Host for testing.
- Uses the current workspace folder as the extension development path.
- Opens the workspace folder explicitly to avoid missing workspace root errors.

## Project folders
- `.vscode`: VS Code debug configuration for the extension.
- `src`: TypeScript source code (extension entry point).
- `dist`: Compiled JavaScript output used by VS Code at runtime.
- `node_modules`: Installed dependencies (generated by npm).

## Running locally
1) `npm install`
2) Press F5 in VS Code to start the Extension Host

## Notes
- API keys must not be hardcoded; use settings/secrets.

## package.json (details)
- "name", "displayName", "description": identite et description de l’extension.
- "publisher": editeur de l’extension.
- "version": version de l’extension.
- "engines.vscode": version minimale de VS Code.
- "categories": classement Marketplace.
- "activationEvents": charge l’extension a l’execution d’une commande.
- "main": point d’entree compile (dist/extension.js).
- "contributes.commands": declaration des commandes VS Code visibles.
- "scripts": scripts npm pour compiler.
- "devDependencies": dependances de developpement TypeScript.
